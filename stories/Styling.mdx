<Title>Styling</Title>

`JBPopover` exposes a compact styling API for anchored popovers and mobile bottom sheets. Use variables for the shell and backdrop, then use the official `content` part for borders, shadows, and the visible content surface.

## Common Design Tokens

Use the core theme docs as the source of truth for shared design-system variables:

- [Theme Colors](../../jb-core/theme/stories/colors.mdx) explains the general color palette variables, default colors, customization approaches, and color generation helpers.
- [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) explains shared size and radius variables.
- [Theme Custom Galleries](https://javadbat.github.io/design-system/?path=/docs/theme-custom-material--docs) gathers the shared custom style recipes across components.

Popover recipes should use those common tokens where possible, then map them into `--jb-popover-*` variables only where the overlay needs a component-specific value.

## Styling Model

Use four layers when styling popover examples:

- Common design tokens are documented in [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx).
- Reusable example theme packs live in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and can be shared across components.
- Component recipe files live in [`modules/jb-popover/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-popover/stories/styles).
- Trigger buttons in these examples reuse [JBButton Styling](../../jb-button/stories/Styling.mdx); the popover recipe only owns the overlay shell and content surface.

Each example uses one shared example theme pack from [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) and one popover-specific recipe from [`modules/jb-popover/stories/styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-popover/stories/styles). The style story applies the theme class to the demo shell and the popover class directly to `JBPopover`.

## Recommended Customization Path

1. Use `--jb-popover-*` variables first for background, radius, padding, mobile backdrop, margin, and z-index.
2. Use `jb-popover::part(content)` for the visible surface: border, shadow, minimum width, overflow, and backdrop filter.
3. Style slotted content with normal CSS classes in the parent document.
4. Reuse [JBButton Styling](../../jb-button/stories/Styling.mdx) for trigger buttons instead of copying button variables into popover recipes.
5. Use common theme variables from [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) when the style should align with the design system.
6. Put reusable theme-pack values in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles) only when they should be shared by multiple component demos.
7. Add a new popover variable only when a needed internal surface cannot be customized with current public variables or official parts.

## Starter Recipe

Use this minimal recipe when a product only needs brand alignment for the popover shell.

```css
jb-popover.brand-popover {
  --jb-popover-bg-color: #ffffff;
  --jb-popover-border-radius: 1rem;
  --jb-popover-padding: 0;
  --jb-popover-margin-top: 0.5rem;
  --jb-popover-back-bg-color: rgb(15 23 42 / 0.45);
  --jb-popover-z-index: 1000;
}

jb-popover.brand-popover::part(content) {
  min-width: 16rem;
  overflow: hidden;
  border: 1px solid #dbe3ef;
  box-shadow: 0 0.75rem 1.75rem rgb(15 23 42 / 0.14);
}
```

## Theme File Pattern

Use [Theme Colors](../../jb-core/theme/stories/colors.mdx) and [Theme Sizes](../../jb-core/theme/stories/sizes.mdx) as the common design-token guides. Theme-pack files should keep reusable demo variables in [`docs/styles`](https://github.com/javadbat/design-system/tree/main/docs/styles), then each component should keep its own style CSS in its story [`styles`](https://github.com/javadbat/design-system/tree/main/modules/jb-popover/stories/styles) folder.

```css
/* https://github.com/javadbat/design-system/blob/main/docs/styles/carbon.css */
.theme-name-style {
  /* shared design-system variables */
}

/* https://github.com/javadbat/design-system/blob/main/modules/jb-popover/stories/styles/style-carbon.css */
jb-popover.theme-name-popover {
  /* popover host variables */
}

jb-popover.theme-name-popover::part(content) {
  /* official content surface */
}

.theme-name-popover-card {
  /* slotted demo content */
}
```

This keeps common theme tokens reusable while making the official `JBPopover` styling API visible in one page.
